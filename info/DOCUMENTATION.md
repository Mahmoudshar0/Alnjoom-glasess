## Authentication & Roles

### JWT Flow
1. Client POST `/api/auth/login` with `{ email, password }`
2. Server validates, returns `{ token, user }`
3. Client stores token, sends as `Authorization: Bearer <token>` on all requests
4. Token expiry: 8 hours

### Roles

**EMPLOYEE** вЂ” Daily operations:
- Customers, Examinations, Orders, Invoices, Inventory
- Customer Reports
- Own profile / settings

**ADMIN** вЂ” Full access:
- Everything Employee can do
- Employee management
- Financial reports
- Backup & restore system

## Deployment & Scripts

### Setup (`setup_v2.bat`)
Full one-click install for a new machine:
1. Checks internet connection
2. Installs Node.js (via winget if missing, refreshes PATH)
3. Installs Git (via winget if missing)
4. Installs PostgreSQL 16 (via winget if missing)
5. Installs PM2 globally (refreshes npm global PATH)
6. Prompts for PostgreSQL password (default: `postgres`)
7. Generates `backend/.env` (DATABASE_URL, JWT_SECRET, PORT)
8. Creates `optivision` database if not exists
9. Installs npm packages (backend + frontend)
10. Runs Prisma generate
11. Runs Prisma migrations
12. Seeds database (admin + staff accounts)

### Start (`start.bat`)
Smart launcher with state detection:
- If already running в†’ control menu (Open / Stop / Restart / Exit)
- If partial state в†’ auto-cleanup then relaunch
- Fresh launch:
  1. Checks Node.js and PM2
  2. Starts PostgreSQL service if stopped
  3. Runs `prisma db push` to sync schema
  4. Builds TypeScript в†’ `dist/index.js`
  5. Generates `ecosystem.config.js` with env vars from `.env`
  6. Starts both servers via PM2 (invisible background)
  7. Opens browser в†’ `http://localhost:5173`

### Update (`update.bat`)
Safe one-click update:
1. Stops PM2 processes + kills ports 3001 and 5173
2. Creates database backup (pg_dump в†’ `backend/backups/`)
3. Runs `git pull`
4. Prompts user to run `start.bat` (or `setup_v2.bat` if deps changed)

## Key Technical Notes

### Prisma 7 Configuration
Prisma 7 removed `url` from `schema.prisma`. Connection is configured in two places:
- **CLI** (migrate, generate, db push): `backend/prisma.config.ts` reads `DATABASE_URL` via dotenv
- **Runtime** (PrismaClient): `backend/src/lib/prisma.ts` uses `@prisma/adapter-pg` (PrismaPg adapter)

### PM2 Environment Variables
PM2 runs as a daemon and does not inherit the CMD session's environment. The `start.bat` generates `ecosystem.config.js` using a Node.js script that reads `backend/.env` directly and embeds `DATABASE_URL`, `JWT_SECRET`, and `PORT` into the PM2 `env` block. `setlocal disabledelayedexpansion` is used around the Node.js command to prevent CMD from consuming `!` characters.

### Auto-backup
The backend schedules automatic pg_dump backups on startup based on settings stored in `backend/backups/settings.json`. Default: weekly, keep last 8.
